# .impeccable.md — AI Kanban Design Context

## Design Context

### Users
Small team of developers using AI-assisted task management as part of their daily workflow. They're comfortable with developer tooling and appreciate interfaces that feel crafted rather than corporate. They spend real time inside this app — it should feel like a place they *want* to be, not just a utility to endure.

### Brand Personality
**Playful, experimental, bold.**

The interface should have character. Not playful in a childish sense — playful in the sense of confident, surprising, and a little irreverent. It should feel like it was made by people who care about craft and aren't afraid to make a strong aesthetic choice.

### Aesthetic Direction
- **Bold visual hierarchy** — strong contrast, confident type weights, clear focal points
- **Experimental edge** — not another gray productivity tool; willing to use unexpected color, unconventional layout choices, or delightful micro-interactions
- **Dark mode as primary** — feels more alive and intentional in dark; light mode is a fallback, not the hero
- **Stage colors should pop** — the blue/orange/purple/green task stages are a core visual identity; lean into them, don't mute them
- **Geist font** — already excellent; use it expressively (tight tracking for headers, bold weights, size contrast)
- **Accessible but not sanitized** — meet WCAG AA but don't let accessibility requirements flatten the design

### Design Principles
1. **Character over convention** — if a safe choice and a bold choice both work, pick bold
2. **Color tells the story** — task stages, priorities, and states should be immediately legible through color; never rely on text alone
3. **Delight in the details** — hover states, transitions, empty states, and micro-copy are all opportunities to show craft
4. **Dense but breathable** — kanban boards need to show a lot; maintain visual rhythm without becoming cluttered
5. **Dark-first** — design decisions should look their best in dark mode; light mode should feel coherent but secondary

### Tech Constraints
- Tailwind CSS 4 + shadcn/ui (Radix primitives) — extend the system, don't fight it
- CSS custom properties in OKLCH — all colors via design tokens in `globals.css`
- dnd-kit for drag interactions — transitions and feedback should feel smooth and intentional
- Mobile-first responsive layout — bottom nav on mobile, sidebar on desktop
